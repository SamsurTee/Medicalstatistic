Updated Medical Statistics Module Pack

This module pack provides fresh, hands-on exercises using synthetic datasets, an AI-assisted teaching activity, and a quiz for your medical statistics course. It was developed after reviewing your existing course materials and researching best practices in synthetic data and AI in education. Evidence suggests that synthetic datasets are valuable for simulation, algorithm testing, public health research and education ([pmc.ncbi.nlm.nih.gov](https://pmc.ncbi.nlm.nih.gov/articles/PMC9931305/#:~:text=products%2C%20services%2C%20or%20systems,publicly%20accessible%20health%20care%20datasets)), while AI tools like ChatGPT can complement traditional teaching by offering additional perspectives and fostering curiosity ([www.statisticsteacher.org](https://www.statisticsteacher.org/2023/11/13/embracing-ai-driven-education/)).

1. Synthetic-Dataset SPSS/R Exercises

Each exercise includes a scenario, dataset description, R code to generate the synthetic data, SPSS instructions, and suggested tasks.

Exercise 1 – Descriptive Statistics & Graphs

Scenario: A researcher measures blood pressure, weight and height for 100 adults.
Variables: Age, Sex, Weight_kg, Height_cm, SystolicBP, DiastolicBP.

R code to generate the data:

set.seed(42)
n <- 100
data1 <- data.frame(
  Age = sample(20:70, n, replace=TRUE),
  Sex = sample(c("Male","Female"), n, replace=TRUE),
  Weight_kg = round(rnorm(n, 70, 12), 1),
  Height_cm = round(rnorm(n, 170, 10), 1),
  SystolicBP = round(rnorm(n, 120, 15), 1),
  DiastolicBP = round(rnorm(n, 80, 10), 1)
)
write.csv(data1, "exercise1.csv", row.names=FALSE)

SPSS instructions: Import the CSV via File → Import Data → CSV. Define variable types, then use Analyze → Descriptive Statistics to compute means and create histograms.

Tasks:

  • Compute summary statistics (mean, SD, quartiles) for each numeric variable.
  • Draw histograms for Weight_kg and SystolicBP and comment on their shapes.
  • Compare average blood pressure by sex.

Exercise 2 – Independent Samples t‑Test

Scenario: A clinical trial compares cholesterol reduction between treatment and control groups.
Variables: Group (Treatment/Control), CholesterolReduction.

R code:

set.seed(42)
n <- 60
Group <- sample(c("Treatment","Control"), n, replace=TRUE)
CholReduction <- ifelse(Group=="Treatment",
                         rnorm(n, 15, 5),
                         rnorm(n, 5, 5))
data2 <- data.frame(Group,
                   CholesterolReduction = round(CholReduction,1))
write.csv(data2, "exercise2.csv", row.names=FALSE)

SPSS instructions: Import the CSV, then use Analyze → Compare Means → Independent Samples t‑Test. Define Group as the grouping variable and CholesterolReduction as the test variable.

Tasks:

  • Test whether the mean cholesterol reduction differs significantly between groups at α=0.05.
  • Report the t‑statistic, degrees of freedom, p‑value and 95 % confidence interval.

Exercise 3 – Chi‑Square Test of Independence

Scenario: Investigate the association between smoking status and disease presence in 120 individuals.
Variables: SmokingStatus (Smoker/NonSmoker), DiseaseStatus (Disease/NoDisease).

R code:

set.seed(42)
n <- 120
SmokingStatus <- sample(c("Smoker","NonSmoker"), n, replace=TRUE, prob=c(0.4,0.6))
DiseaseStatus <- mapply(function(sm){
  p <- ifelse(sm=="Smoker", 0.6, 0.3)
  sample(c("Disease","NoDisease"), 1, prob=c(p, 1-p))
}, SmokingStatus)
data3 <- data.frame(SmokingStatus, DiseaseStatus)
write.csv(data3, "exercise3.csv", row.names=FALSE)

SPSS instructions: Import the CSV and use Analyze → Descriptive Statistics → Crosstabs with Chi‑square option.

Tasks:

  • Construct the contingency table and compute marginal totals.
  • Perform a chi‑square test for independence (α=0.05).
  • Interpret the chi‑square statistic, degrees of freedom and p‑value; compute the odds ratio.

Exercise 4 – One‑Way ANOVA

Scenario: Compare BMI among participants following diets A, B and C.
Variables: DietGroup (DietA/DietB/DietC), BMI.

R code:

set.seed(42)
n <- 90
DietGroup <- sample(c("DietA","DietB","DietC"), n, replace=TRUE)
BMI <- sapply(DietGroup, function(d){
  if(d=="DietA") { rnorm(1, 25, 3) }
  else if(d=="DietB") { rnorm(1, 27, 3) }
  else { rnorm(1, 24, 3) }
})
data4 <- data.frame(DietGroup, BMI = round(BMI,1))
write.csv(data4, "exercise4.csv", row.names=FALSE)

SPSS instructions: Import the CSV, then use Analyze → Compare Means → One‑Way ANOVA with DietGroup as the factor. Request Tukey post‑hoc tests.

Tasks:

  • State the null and alternative hypotheses and perform ANOVA.
  • Report the F‑statistic, degrees of freedom and p‑value.
  • Interpret which groups differ using post‑hoc comparisons.
  • Check assumptions (normality, equal variances).

Exercise 5 – Correlation & Simple Linear Regression

Scenario: Explore the relationships among age, BMI, systolic blood pressure and heart rate in 80 adults.
Variables: Age, BMI, SystolicBP, HeartRate.

R code:

set.seed(42)
n <- 80
Age <- sample(30:75, n, replace=TRUE)
BMI <- round(rnorm(n, 26, 4),1)
SystolicBP <- round(90 + 0.8*Age + rnorm(n, 0, 10),1)
HeartRate <- round(70 - 0.2*Age + rnorm(n, 0, 5),1)
data5 <- data.frame(Age, BMI, SystolicBP, HeartRate)
write.csv(data5, "exercise5.csv", row.names=FALSE)

SPSS instructions: Import the CSV. Use scatter plots to visualise relationships, then Analyze → Correlate → Bivariate for correlations and Analyze → Regression → Linear to regress SystolicBP on Age (and optionally BMI).

Tasks:

  • Compute the correlation matrix and identify significant associations.
  • Fit a simple linear regression predicting SBP from age; report coefficients and R².
  • Check residual plots for assumptions.
  • Optionally include BMI in a multiple regression.

Exercise 6 – Logistic Regression

Scenario: Examine how age, sex, cholesterol and treatment status predict the occurrence of a cardiovascular event in 100 individuals.
Variables: Age, Sex, Cholesterol, Treatment, Event.

R code:

set.seed(42)
n <- 100
Age <- sample(40:80, n, replace=TRUE)
Sex <- sample(c("Male","Female"), n, replace=TRUE)
Chol <- rnorm(n, 200, 30)
Treatment <- sample(0:1, n, replace=TRUE)
logit <- -8 + 0.05*Age + 0.01*Chol - 0.5*Treatment
prob <- 1/(1+exp(-logit))
Event <- rbinom(n, 1, prob)
data6 <- data.frame(Age, Sex, Cholesterol=round(Chol,1),
                    Treatment, Event)
write.csv(data6, "exercise6.csv", row.names=FALSE)

SPSS instructions: Import the CSV, then use Analyze → Regression → Binary Logistic with Event as the dependent variable and Age, Sex, Cholesterol, Treatment as predictors.

Tasks:

  • Interpret regression coefficients and compute odds ratios with confidence intervals.
  • Assess model fit (pseudo‑R², Hosmer–Lemeshow test).
  • Predict the probability of an event for a 60‑year‑old male with cholesterol 220 mg/dL under treatment vs control.

Exercise 7 – Survival Analysis (Kaplan–Meier & Cox)

Scenario: Analyse time‑to‑event data for 120 patients with right censoring.
Variables: Time (years), Event (1/0), Treatment (0/1), Age.

R code:

set.seed(42)
n <- 120
Treatment <- sample(0:1, n, replace=TRUE)
Age <- sample(50:85, n, replace=TRUE)
base_rate <- 0.05
times <- numeric(n)
events <- numeric(n)
for(i in 1:n){
  rate <- base_rate * (0.7^Treatment[i]) * (1.02^(Age[i]-50))
  t <- rexp(1, rate)
  censor <- runif(1, 5, 10)
  if(t <= censor){
    times[i] <- round(t,2); events[i] <- 1
  } else {
    times[i] <- round(censor,2); events[i] <- 0
  }
}
data7 <- data.frame(Time=times, Event=events, Treatment, Age)
write.csv(data7, "exercise7.csv", row.names=FALSE)

SPSS instructions: Import the CSV. Use Analyze → Survival → Kaplan–Meier to plot survival curves by treatment and perform a log‑rank test. For Cox regression, use Analyze → Survival → Cox Regression with Treatment and Age as covariates.

Tasks:

  • Plot Kaplan–Meier curves for treatment vs control and interpret differences.
  • Run the log‑rank test for equality of survival functions.
  • Fit a Cox proportional hazards model and interpret hazard ratios for Treatment and Age.
  • Check the proportional hazards assumption.

2. AI-Assisted Teaching Activity

This activity encourages students to use generative AI as a collaborator while developing critical evaluation skills.

  1. Pre‑reading discussion: Provide readings on AI in education. Discuss benefits (additional resources and perspectives) and concerns about overreliance ([www.statisticsteacher.org](https://www.statisticsteacher.org/2023/11/13/embracing-ai-driven-education/)). Highlight frameworks for healthy skepticism ([teaching.virginia.edu](https://teaching.virginia.edu/collections/generative-ai-in-statistics#:~:text=Generative%20AI%20in%20Statistics%20Generative,powered%20solutions.%20Rich%20Ross)).
  2. Prompt‑writing workshop: Students work in pairs to craft prompts asking an AI tool to generate R/SPSS code for one of the exercises. They refine prompts to improve clarity and completeness.
  3. Execution & verification: Students run the AI‑generated code, compare it to their own code or textbook solutions, and verify accuracy and adherence to statistical principles.
  4. Critical reflection: Groups share what worked, note any AI mistakes and discuss how they identified and corrected them.
  5. Ethics & synthetic data discussion: Discuss why synthetic data are useful (simulation, algorithm testing, education) and limitations such as potential data leakage and model dependence ([pmc.ncbi.nlm.nih.gov](https://pmc.ncbi.nlm.nih.gov/articles/PMC9931305/)).

3. Quiz

The 25‑question quiz assesses students’ understanding of core concepts. Questions include definitions of descriptive vs inferential statistics, assumptions for t‑tests, interpretation of p‑values, choice of non‑parametric tests, chi‑square test concepts, ANOVA hypotheses and F‑statistics, post‑hoc reasoning, correlation and regression interpretation, logistic and survival analysis metrics, advantages and limitations of synthetic data, and responsible use of AI in statistics education. (See the accompanying document or teaching notes for the complete question list and answers.)

4. Implementation Notes

  • Dataset distribution: Provide the generated CSV files or allow students to run the provided R scripts to create them. Synthetic datasets protect privacy and support educational use ([pmc.ncbi.nlm.nih.gov](https://pmc.ncbi.nlm.nih.gov/articles/PMC9931305/#:~:text=products%2C%20services%2C%20or%20systems,publicly%20accessible%20health%20care%20datasets)).
  • Software flexibility: Offer both SPSS and R instructions to build cross‑platform skills.
  • AI activity timing: Introduce the AI‑assisted activity once students have basic programming proficiency.
  • Assessment: Use the quiz as a formative or summative assessment; mix conceptual and applied questions.
  • Alternative resources: Consider using public synthetic datasets (e.g., Health Gym or Synthea) or other AI tools for variation.
